# phase2_1_tech_stack.md (updated)

## Frontend

**Framework / Library:** React (latest stable)  
- Same justification as before  
- Additional usage: Admin GUI form

**Styling:** Tailwind CSS  
- Same justification  
- Applied to admin form layout

**Routing / Navigation:** React Router  
- Includes routes for admin pages:
  - /admin/add-game
  - /admin/edit-game/:id
  - /admin/archived

---

## Backend / Static Site Generation

**Static Site Generator:** Next.js (React-based, SSG mode)  
- Same justification  
- Additional API routes for admin actions:
  - `/api/admin/add-game`
  - `/api/admin/update-game`
  - `/api/admin/archive-game`
- API routes encapsulate GitHub interactions
- Design principle: **GitHub-specific logic isolated in a single service layer**  
  - Future alternative hosting or storage can replace this service without changing frontend or SSG logic

**Data Format:** JSON or Markdown for game data  
- Same justification  
- Admin GUI generates JSON/Markdown files via API

---

## GitHub Integration (Admin GUI)

**API:** GitHub REST or GraphQL  
- Handles:
  - File creation/update (JSON/Markdown)
  - Image upload
  - Commit creation with meaningful messages
- Admin API routes call GitHub service
- Security:
  - Personal Access Token or GitHub App stored securely
  - Only admin can trigger actions

**Modularity:**  
- GitHub service encapsulated in a single module
- Interface: `addGame(data)`, `updateGame(id, data)`, `archiveGame(id)`  
- Any future backend or alternative repo can implement the same interface

---

## Hosting / Deployment

- Render static hosting remains unchanged
- Admin GUI interacts with serverless API routes (Next.js)
- Build triggered automatically on repo update

## Rationale Summary

| Requirement | Choice | Why |
|-------------|--------|-----|
| Responsive frontend + admin GUI | React + Tailwind | Large ecosystem, simple form creation, mobile/tablet-friendly |
| Static content + admin updates | Next.js SSG + API routes | Generates static site, enables controlled admin commits |
| Data storage | JSON / Markdown | Simple, human-readable, versionable, generated by admin GUI |
| Hosting | Render | Free, automatic rebuild on repo push |
| Git integration | GitHub API encapsulated in service | Isolated dependency, easily replaceable in future |
| Documentation | Markdown + inline | Fully explicit, maintainable, includes admin GUI workflow |
